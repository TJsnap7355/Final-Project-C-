# Contributing

Thank you for considering contributing to this project!

## How to Contribute

1. Fork the repository
2. Create a new branch (`feature-name`)
3. Make your changes
4. Submit a pull request

## Contribution Requirements

To ensure consistency and maintainability, all contributions must follow these guidelines:

- Code must be written in **C++ (C++14 standard)**
- Follow consistent indentation and formatting
- Use clear and descriptive variable and function names
- Avoid global variables when possible
- Include comments explaining important logic
- Ensure the program compiles without errors
- Do not include compiled files (.exe, build folders)

## Code Style

- Use camelCase for variables and functions
- Use meaningful names (avoid single-letter variables unless necessary)
- Keep functions modular and focused on one task

## Pull Request Guidelines

- Clearly describe what your changes do
- Keep pull requests focused on one feature or fix
- Test your code before submitting
